B. 자동화 게이트 (Trigger)

5 항목
"작업완료" → 자동 commit+push : confirm 1회 추가 harness-hardening-v1
① 사용자 첫 발화 ("작업완료") → Claude 가 echo: git status / git diff --name-only / git log origin/master..HEAD
② push 대상 + 변경 보여주고 confirm 요청
③ 진행 신호 (yes / y / 응 / ok / 동일 트리거 재발화) → 진행
④ 다른 응답 (잠깐만 / 취소 / 추가 질문) → cancel + 일반 대화 모드
Why — 단발 발화 즉시 push 시 의도치 않은 commit 위험. confirm 1회로 사용자 검증 게이트
git add 명시 파일만 — -A / . 금지
commit 시 변경 파일을 정확히 지정. 일괄 추가 시 .env / credential / target/ 등 의도치 않은 파일 포함 위험
저위험 트리거 (single-shot OK)
"집이야" / "사무실이야" / "출장중이야" → dev-environments private repo push
시크릿 마스킹 검증 fail 시 자동 cancel
Why — 환경 갱신은 자주 + 안전 (private repo + 마스킹 검증) → confirm 부담 제거
금지 commit 패턴 위반 시 차단
.env · credential 파일 · 대용량 바이너리 · 민감 파일 (CSV 백업 등)
빠른모드 opt-in
TRIGGER_FAST_MODE=1 env set 시 confirm 단계 skip (echo 는 유지). 신뢰된 일괄 작업용
