목적 인프라 DB (InfraServer) 값 ↔ 현장 수집값(InspectMetricSnapshot.raw_payload) 비교 → 차이 시 ⚠ 알림. 액션 없음 (자동 수정 X, 저장 차단 X) — 사용자 인지 + 수동 수정 유도.
확정된 핵심 FR (FR-4·5·6·7)
  • 알림 시점 = 페이지 로드 + 저장 시 재확인
  • 알림 표현 = 인라인 ⚠ + 상단 요약 박스 (둘 다)
  • 액션 없음 / dismiss 가능
  • 비교 로직 = 프론트 JS (백엔드 모델 변경 0)
DB팀 자문 필요 (5건)
  • raw_payload JSON schema 매핑
  • hostname 컬럼 위치 (InfraServer 에 없음)
  • 사용자 수동 입력 form 필드 포함 여부
  • InspectCheckResult 도 비교 source?
  • 다중 호스트 매칭 키
디자인팀 자문 필요 (4건)
  • 알림 박스 위치 (3안 시안)
  • 인라인 ⚠ + 툴팁 구현 방식
  • 색 강도 (`--warning` vs `--scope-std` 톤)
  • dismiss UX (localStorage / 세션 / 매번)
👇 사용자 검토 액션: (i) 큰 방향 OK → DB팀 + 디자인팀 자문 시작 · (ii) 수정/추가 요구사항 있으면 알려주세요 · (iii) FR-9 (한글 라벨) / FR-10 (정규화 룰) / R-2 (단계적 도입) 세부 의견
