# 절사(ROUNDDOWN) 규칙 가이드

## 정의

절사란 특정 자릿수 이하의 금액을 버리는(내림) 처리를 말한다.
명칭은 "X단위 절사"로 표기하며, 해당 단위 이하의 금액이 잘려나간다.

## 절사 단위별 규칙

| 명칭 | 의미 | 계산 단위값 | 예시 |
|------|------|------------|------|
| 절사 없음 | 절사 미적용 | 1 | 5,550원 → 5,550원 |
| 백단위 절사 | 백원 이하 절사 | 1,000 | 5,500원 → 5,000원 |
| 천단위 절사 | 천원 이하 절사 | 10,000 | 55,000원 → 50,000원 |
| 만단위 절사 | 만원 이하 절사 | 100,000 | 550,000원 → 500,000원 |
| 십만단위 절사 | 십만원 이하 절사 | 1,000,000 | 5,500,000원 → 5,000,000원 |

## 계산 공식

```
절사결과 = Math.floor(금액 / 계산단위값) * 계산단위값
```

예시: 백단위 절사 (계산단위값 = 1000)
- 5,500 → Math.floor(5500 / 1000) * 1000 = 5 * 1000 = 5,000

## 적용 범위

- 절사는 **최종 합계 금액에만** 적용한다.
- 소계(공급가액), 부가세에는 절사를 적용하지 않는다.
- 낙찰율 적용 후에도 절사가 재적용된다.

## 계산 흐름

```
1. rawTotal = 품목별 금액 합산
2. VAT 포함 여부 판단
   - VAT 포함: total = rawTotal
   - VAT 미포함: total = rawTotal + (rawTotal * 10 / 100)
3. 절사 적용: total = roundDown(total, 계산단위값)
4. 낙찰율 적용 (있는 경우): total = Math.floor(total * 낙찰율 / 100)
5. 절사 재적용: total = roundDown(total, 계산단위값)
```

## 중요 사항

- "백단위 절사"는 "백원 단위로 자른다"가 아니라 "백원 자릿수를 버린다(천원 단위로 남긴다)"는 의미이다.
- DB에 저장되는 rounddown_unit 값은 **계산 단위값**이다 (백단위=1000, 천단위=10000, 만단위=100000, 십만단위=1000000).
- UI에서는 "백단위 절사", "천단위 절사" 등 사용자 친화적 명칭을 사용한다.
