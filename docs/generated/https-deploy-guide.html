1

iptime DDNS 등록

직접 — 공유기 관리자 웹 UI
  • iptime 관리자(192.168.0.1) 접속 → 고급설정 → 특수기능 → DDNS
  • 호스트이름 xxxx.iptime.org 등록, 사용자 계정 입력 → 정상 갱신 상태 확인
이 도메인이 곧 외부에서 접속할 주소입니다. 위 입력칸에 같은 값을 넣어두세요.
2

포트포워딩 + Tomcat 8080 외부차단

직접 — 공유기 + Windows 방화벽
  • iptime 포트포워딩: 외부 80 → 서버 80, 외부 443 → 서버 443 (80은 win-acme http-01 챌린지에 필요)
  • 8080 외부차단 — 둘 중 하나:
① Windows Defender 방화벽 인바운드 규칙으로 8080 외부 차단 (PowerShell, 관리자):
New-NetFirewallRule -DisplayName "Block-Tomcat-8080-External" -Direction Inbound -LocalPort 8080 -Protocol TCP -Action Block -RemoteAddress Internet
② 또는 Tomcat conf/server.xml 의 8080 Connector 에 address="127.0.0.1" 추가 (로컬 바인드 제한).
차단 방식을 기억해두세요 — 롤백 절차가 방식별로 다릅니다(맨 아래).
3

Windows nginx 설치

직접 — 설치
  • nginx for Windows 를 C:\nginx\ 에 설치
  • 서비스 등록(nssm/winsw)으로 부팅 자동시작 권장
  • ACME 챌린지용 webroot 폴더 생성: C:\nginx\html\.well-known\acme-challenge\
New-Item -ItemType Directory -Force C:\nginx\html\.well-known\acme-challenge | Out-Null
4

nginx 80 챌린지 블록 먼저 적용

제가 작성 — 발급 전 80 응답 필수
순서 중요: 443 인증서가 아직 없으므로 80 블록만 먼저 올립니다. http-01 챌린지가 80에서 응답해야 발급됩니다.
C:\nginx\conf\nginx.conf 의 http { } 안에 추가:
server {
  listen 80;
  server_name xxxx.iptime.org;
  location /.well-known/acme-challenge/ { root C:/nginx/html; }   # 챌린지 (redirect보다 우선)
  location / { return 301 https://$host$request_uri; }            # http -> https
}
설정 검증 후 서비스 시작:
C:\nginx\nginx.exe -t
# 이상 없으면 nginx 서비스 start (nssm start nginx 또는 C:\nginx\nginx.exe)
  • 외부망(휴대폰 등)에서 http://xxxx.iptime.org/.well-known/acme-challenge/ping 접속 → 80 도달(404라도 nginx 응답) 확인
5

win-acme 로 인증서 발급

직접 — wacs.exe 실행
먼저 staging 리허설(운영 rate-limit 무소모, 전용 경로):
C:\tools\win-acme\wacs.exe --test --notaskscheduler --source manual --host xxxx.iptime.org --validation filesystem --webroot C:\nginx\html --store pemfiles --pemfilespath C:\nginx\ssl-staging
리허설 성공하면 staging 폴더 삭제: Remove-Item -Recurse -Force C:\nginx\ssl-staging · staging 인증서는 신뢰되지 않으니 절대 운영 경로에 두지 마세요.
실제 운영 발급(production):
C:\tools\win-acme\wacs.exe --source manual --host xxxx.iptime.org --validation filesystem --webroot C:\nginx\html --store pemfiles --pemfilespath C:\nginx\ssl
  • 발급 후 C:\nginx\ssl 의 실제 PEM 파일명 확인 (Step 6 경로에 반영)
  • win-acme 가 자동갱신 예약작업(Task Scheduler) 생성 — post-renewal 로 nginx -s reload 등록 권장
6

nginx 443 TLS 블록 추가 + reload

제가 작성
Step 4 의 80 블록 아래에 추가 (PEM 파일명은 Step 5 실제 값으로):
server {
  listen 443 ssl;
  server_name xxxx.iptime.org;
  ssl_certificate     C:/nginx/ssl/xxxx.iptime.org-chain.pem;   # 실제 파일명으로
  ssl_certificate_key C:/nginx/ssl/xxxx.iptime.org-key.pem;     # 실제 파일명으로
  ssl_protocols       TLSv1.2 TLSv1.3;

  client_max_body_size 50m;   # 기본 1MB -> 업로드 413 방지 (앱 한도에 맞춰 조정)
  add_header Strict-Transport-Security "max-age=300" always;  # 초도 짧게, 안정화 후 상향

  location / {
    proxy_pass         http://127.0.0.1:8080;
    proxy_http_version 1.1;
    proxy_read_timeout 300s;
    proxy_set_header   Host              $host;
    proxy_set_header   X-Real-IP         $remote_addr;
    proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
    proxy_set_header   X-Forwarded-Proto https;
    proxy_set_header   X-Forwarded-Host  $host;
    proxy_set_header   X-Forwarded-Port  443;
  }
}
C:\nginx\nginx.exe -t
C:\nginx\nginx.exe -s reload
7

앱 설정 1줄 + ROOT.war 재배포

제가 편집 가능
.properties 는 값 뒤 주석 금지 — 주석은 반드시 별도 줄로. (인라인 주석 쓰면 값이 통째로 잘못 파싱됨)
application-prod.properties 에 추가:
server.forward-headers-strategy=framework
  • ROOT.war 재배포 (배포 절차는 기존 운영 배포 메모 그대로)
이 한 줄 편집·재배포는 제가 해드릴 수 있습니다 — 말씀만 주세요.

✅ 검증 체크리스트 (프록시 443 경유로)

↩ 롤백 (문제 시)

  • 무중단: nginx 서비스 중지 → Tomcat(8080) 그대로. 단 8080 외부차단 했으면 아래 방식별 복귀 필요
  • 방화벽으로 차단한 경우 → Defender 규칙 비활성화 + iptime 포트포워딩(외부 8080→서버 8080) 임시 복구
  • Connector address=127.0.0.1 로 차단한 경우 → address 제거 + Tomcat 재기동 + 포트포워딩 복구
  • properties 동반 롤백: HTTP 로 완전 복귀 시 forward-headers-strategy 제거 + cookie.secure=false 로 재배포
⚠ *.iptime.org 공유 도메인은 Let's Encrypt rate-limit 공유 위험 — 장기 운영이면 본인 도메인 권장. TLS 본격 하드닝(cipher/OCSP)은 별도 작업.
