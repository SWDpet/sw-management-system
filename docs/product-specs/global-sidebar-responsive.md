## 0. 한 줄 요약

전 화면(약 52개)의 **상단 GNB → 좌측 사이드바**로 통일하고, **1920×1080 기준 + 고해상도 + 태블릿 + 모바일 반응형**을 적용한다. 동시에 `/dashboard-preview`를 **메인 대시보드(`/`)로 승격**하고, 데이터 카테고리 색을 **CSS 토큰화**한다.

## 1. 배경 / 목적

- 현재 상단 GNB는 메뉴가 많아(서버/사업/담당자/라이선스▾/견적서▾/업무계획/문서▾/성과/관리자▾) 가로로 빽빽하고, 좁은 화면에서 깨진다.
- 사이드바는 **메뉴 그룹화·확장성·데이터 영역 확보**에 유리(Linear/Notion/Attio류 B2B 앱 셸).
- **반응형 미흡**: 현재 데스크탑 고정 폭 위주 → 태블릿·모바일·고해상도 대응 필요.

## 2. 범위 (In / Out)

**In**
- **공통 사이드바 프래그먼트** 신규(`fragments/sidebar.html` 또는 `top-nav.html` 대체) — 51개 `th:replace` 화면 + main-dashboard 인라인 1개.
- **반응형**: 모바일/태블릿/데스크탑(1920 기준)/와이드 4단 브레이크포인트.
- **대시보드 승격**: `/dashboard-preview` 레이아웃·기능을 `/`(main-dashboard)로 이관.
- **색 토큰화**: 데이터 카테고리색 `--cat-1`~`--cat-6` 를 `design-system.css` 에 정의.
- 기존 메뉴 권한 분기(`authInfra`/`authProject`/… `sec:authorize`) 100% 보존.

**Out**
- 다크모드 (사용자 결정 — 제외).
- 각 화면 내부 콘텐츠 재설계(테이블/폼 등)는 본 건 Out — **셸(내비+레이아웃 골격)과 반응형만**. (대시보드 콘텐츠만 예외로 승격)
- 신규 기능/데이터 추가 없음.

## 3. 사용자 스토리

> 사용자로서, 어느 화면에서든 **좌측 사이드바**로 일관되게 이동하고, **PC(1920)·노트북·태블릿·폰** 어디서나 깨지지 않고 쓰고 싶다.

## 4. 기능 요구사항 (FR)

| ID | 요구사항 |
|---|---|
| FR-1 | **공통 사이드바 프래그먼트**: 로고 + 메뉴 그룹(개요/사업/자산/계정) + 하단 사용자·마이페이지·로그아웃. 기존 GNB의 **모든 메뉴·드롭다운 항목·권한 분기** 동일 이전. |
| FR-2 | **`position:fixed` 사이드바 + 본문 오프셋**: 사이드바 고정, 본문은 좌측 여백(데스크탑 230px). **셸 클래스** `app-shell`/`content-shell` 도입 + **예외 클래스** `no-sidebar`/`fullscreen`/`print` — 모달·팀모니터 풀스크린·문서 preview/인쇄·fixed alert 가 오프셋/사이드바에 영향받지 않도록. 전역 CSS로 오프셋. |
| FR-3 | **활성 메뉴 표시**: 현재 경로 **prefix 매칭**(`/projects/*`,`/workplan/*`,`/document/*`,`/ops-doc/*`,`/license/*`,`/geonuris/*`,`/quotation/*`,`/admin/*`,`/infra/*`,`/performance/*`) 으로 사이드바 항목 active. |
| FR-4 (반응형) | 사이드바 거동(확정): **데스크탑(≥1200) 펼침 230px / 태블릿(768–1199) 아이콘 축소(~64px) / 모바일(<768) 햄버거 드로어(오버레이+백드롭)**. |
| FR-5 (반응형) | 본문: 그리드·카드·차트가 화면폭에 따라 열 수 자동 축소(5→3→2→1). 테이블은 좁은 화면에서 가로 스크롤. |
| FR-6 (대시보드 승격) | `/dashboard-preview` 의 구조·실데이터·완료/진행 증감을 `/`(main-dashboard)로 이관. **`/dashboard-preview` → `/` 302 리다이렉트**(확정, 회귀 최소화). preview 내 하드코딩 `/dashboard-preview` 링크·form action 을 `/` 로 치환. `MainController` 의 `/` 모델 속성을 preview 로직(증감·연도추이·금액Top·피드)으로 보강. |
| FR-7 (토큰화) | 현재 `design-system.css` 엔 `--cat-*` 없음 → **신설**(`--cat-1`~`--cat-6`). preview·main-dashboard·차트·막대·증감의 **raw hex 전부 토큰으로 치환**(수용 기준). |
| FR-8 | 모든 화면 권한·세션·기존 기능 **회귀 0**. |

## 5. 반응형 설계 (🎨 디자인팀 자문 대상)

### 5-1. 브레이크포인트
| 구간 | 폭 | 사이드바 | 본문 |
|---|---|---|---|
| 모바일 | `< 768px` | 숨김 + 상단 햄버거 → 드로어(오버레이) | 1열 스택, 테이블 가로 스크롤 |
| 태블릿 | `768–1199px` | **아이콘 축소(~64px)** | 2열 |
| **데스크탑(기준)** | `1200–1919px` | 펼침 230px | 대시보드 KPI 5 / 차트 3 / 피드 3 |
| 와이드 | `≥ 1920px` | 펼침 | 본문 **max-width 1760px 중앙 정렬**(확정, 과도한 늘어짐 방지) |

- **설계 기준 해상도 = 1920×1080** (데스크탑 기준에서 1px 단위로 검수).

### 5-2. 🎨 디자인팀 가상 자문
- 사이드바·셸 색은 **teal/stone 토큰**만(다크모드 제외, 추후 토큰화로 확장 여지). 신규 :root self-ref 금지.
- 데이터 카테고리색 6종 토큰화(FR-7) → 라이트/추후 다크 일관.
- 접근성: 사이드바 `<nav aria-label>`, 모바일 드로어 `aria-expanded`/포커스 트랩/Esc 닫기, 활성 `aria-current=page`.
- 터치 타깃 ≥ 44px(모바일), 가로 스크롤 테이블에 스크롤 힌트.
- ⚠ 판정: 대규모 UI 변경이나 **기존 토큰 재사용 + 공통 프래그먼트 1곳 중심**이라 색/토큰 리스크는 낮음. 실제 리스크는 **51개 화면 레이아웃 회귀**(§7).

## 6. 마이그레이션 전략

- **공통 프래그먼트 중심**: `top-nav.html`(51개 공유)을 사이드바 셸로 교체 → 한 곳 수정이 51개에 전파.
- **본문 오프셋은 전역 CSS**: 사이드바 `position:fixed` + `body`/공통 컨테이너에 `padding-left` (반응형으로 0~230px). 화면별 개별 구조 변경 최소화.
- **인라인 GNB 1개**(main-dashboard) → 승격(FR-6) 시 공통 프래그먼트 사용으로 정리.
- **단계(Phase)**:
  1. 공통 사이드바 프래그먼트 + 반응형 셸 CSS + 토큰 구축
  2. 대시보드 승격(파일럿) → 검증
  3. 나머지 50개 화면 일괄 적용(프래그먼트 전파 + 화면별 컨테이너 오프셋/스크롤 점검)
  4. 화면별 반응형 육안 QA(특히 넓은 테이블·견적서·문서 폼)

## 7. 회귀 / 리스크

- **최대 리스크 = 51개 화면 레이아웃 회귀**: 화면마다 컨테이너 클래스(.container/.admin-container 등)·고정폭·인라인 style 제각각 → 오프셋/반응형 적용 시 깨질 수 있음. → Phase 4 화면별 QA 필수.
- 기존 권한 분기(sec:authorize) 누락 시 메뉴 노출 오류 → FR-1에서 1:1 이전 검증.
- 모바일 드로어 JS·포커스 트랩 신규.
- 다크모드 제외이나, 토큰화는 추후 다크 확장 대비.

## 7-1. QA 수용 기준 (codex — 개발계획서 이관)
- **뷰포트 대표 4종 이상**: `375`(모바일) / `768`(태블릿 하단) / `1199`(태블릿 상단) / `1920`(데스크탑 기준) — 필요 시 `1199/1200` 경계, `2560`(고해상도) 추가.
- **계정 3종**: 관리자 / 일반(EDIT) / 권한 제한(일부 NONE) — 사이드바 메뉴 노출·권한 분기 일치.
- **51개 top-nav 화면** 전수 + main-dashboard: 사이드바 표시·활성·오프셋·콘텐츠 안 깨짐.
- **특수 화면 점검**: 문서/견적서 **PDF·인쇄**, **팀모니터 풀스크린**, 넓은 **금액 테이블**(가로 스크롤), **모달**, **fixed alert/토스트** — `no-sidebar/fullscreen/print` 예외 적용 확인.
- 합격 기준: 위 매트릭스에서 레이아웃 깨짐·메뉴 오노출·가로 overflow 0.
