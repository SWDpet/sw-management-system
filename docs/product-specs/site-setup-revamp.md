# 기획서 — 점검 에이전트 사이트 정보 개편 (코드 자동조합 + 수집대상 선택 + GUI)

> 작성: 2026-05-31 · 워크플로우: 요청 → **기획서(본문서)** → DB·디자인팀 자문(반영) → ~~codex(속도모드 생략)~~ → 사용자 승인 → 구현
> 선행: maint_type 점검범위 분기 Step 1~5 완료(`f923fbf`·`afd056e`·`782a957`).

## 1. 배경 / 목표

setup GUI 의 사이트 정보가 **전부 자유 입력**(지자체명·시스템명·사이트코드 ascii)이라:
- 사이트 코드(`gangjin`)가 임의 → **같은 서버에 UPIS+다른 시스템 공존 시 코드 충돌**(`site.{code}.json` 중복).
- 지자체명/시스템명 오타·표기 불일치 → 서버 `site_code` 매핑 실패(`SiteNotMappedException` 422).

**목표**
1. 시도→시군구→시스템(한글) 선택만으로 **사이트 코드 자동 조합** (충돌 불가, 표기 일치).
2. setup GUI 에서 **수집 대상 섹션 선택**(Step 5 `sections` 연동).
3. GUI **디자인 개편**.
4. 에이전트는 **폐쇄망** → 코드 목록을 **패키지 번들(codes.json)** 로 공급.

## 2. 확정 결정 (2026-05-31 사용자 승인)

| # | 항목 | 확정 |
|---|------|------|
| D1 | 사이트 코드 형식 | **`{adm_sect_c}_{시스템영문}`** — 행정구역코드(시도2+시군구3, 5자리) + `_` + sys_nm_en |
| D2 | 폐쇄망 목록 공급 | **패키지에 `codes.json` 번들 동봉** (빌드 시 sigungu_code + sys_mst 추출) |
| D3 | 섹션 기본값 | **전부 체크(AP/DB/DBMS/GIS) 후 가감** |
| D5 | 기존 코드 | **신코드 전환 + 별칭 유지** (기존 QR/데이터 호환) |
| D4 | 서버 매핑 | (기술) 신코드를 `sw_pjt.site_code` 에 정합 — 아래 §6 |

**근거 데이터(DB 확인):**
- `sigungu_code.adm_sect_c` = **전 행 5자리**(시도2+시군구3). 강진=`46810`, 단양=`43800`.
- `sys_mst`: `cd`=영문(UPIS), `nm`=한글(도시계획정보체계) → 한글 선택 시 영문 자동.
- `sw_pjt.site_code` = VARCHAR(32), UNIQUE(partial). `InspectQrBatch.site_code` = VARCHAR(32). `46810_UPIS`(10자) 여유.
- 서버: `findBySiteCode(payload.site)` → 미스 시 422 + 운영자 보정.

## 3. 사이트 코드 규칙

```
site_code = adm_sect_c + "_" + sys_nm_en        (대문자 시스템, 행정코드 5자리)
예) 강진군 UPIS = 46810_UPIS   강진군 KRAS = 46810_KRAS   단양군 UPIS = 43800_UPIS
```
- **충돌 해소**: 같은 PC라도 시스템영문이 달라 코드 분리 → `site.46810_UPIS.json`, `site.46810_KRAS.json`.
- 길이: 5+1+≤10 = ≤16자 → `site_code` VARCHAR(32) 안전.
- payload.site / 파일명 / sw_pjt.site_code 3자 동일 규칙 → end-to-end 일치.

## 4. 폐쇄망 코드 번들 (`config/codes.json`)

빌드/배포 시 운영 DB 에서 추출해 에이전트 패키지에 동봉. setup GUI 가 이 파일만 읽음(인터넷·DB 불필요).

```json
{
  "generated_at": "2026-05-31",
  "sido": ["전라남도", "충청북도", ...],
  "sigungu": [
    { "sido": "전라남도", "sgg": "강진군", "adm": "46810" },
    { "sido": "충청북도", "sgg": "단양군", "adm": "43800" }
  ],
  "systems": [ { "ko": "도시계획정보체계", "en": "UPIS" }, { "ko": "부동산종합공부시스템", "en": "KRAS" } ]
}
```
- 추출 스크립트: `tools/export-codes.ps1`(또는 서버 측 export). 갱신은 패키지 재배포 시.
- 시군구는 sido 로 필터(종속 드롭다운). 시스템은 ko 표시 / en 저장.

## 5. setup GUI 개편 (🎨 디자인팀 자문 반영)

### 5-1. 동작 명세 (그룹 1 "사이트 정보")
| 컨트롤 | 변경 |
|--------|------|
| 시도 | **콤보**(codes.json sido) |
| 시군구 | **콤보**(선택 시도로 필터) — 종속 |
| 시스템 | **콤보**(systems.ko 표시) |
| 시스템영문 | **자동 표시(readonly)** — 선택 ko→en |

| 사이트 코드 | **자동 조합(readonly)** — `adm_시스템영문`, 회색 강조칸 |

### 5-2. 신규 그룹 "수집 대상"

## 6. 서버 연동 (D4 / D5)

- **정합**: 사업(sw_pjt) 의 `site_code` 를 동일 규칙 `adm_시스템영문` 으로 설정해야 에이전트 payload.site 와 매칭.
  - 사업관리 화면에서 site_code **수동입력 → 자동생성 버튼**(시군구+시스템 선택 시 동일 규칙) 추가 권장(후속).
- **별칭(D5)**: 기존 `gangjin`/`gj`/`dyg` 로 적재된 과거 데이터 호환 위해 `site_code_alias`(또는 매핑 테이블) 유지 — `findBySiteCode` 가 본코드+별칭 모두 조회.
- 미스 시 기존대로 422 + "사업관리에서 site_code 설정" 안내(변경 없음).

## 7. 개발 단계

1. **codes.json 추출** — export 스크립트 + 패키지 동봉(폐쇄망).
2. **setup GUI 개편** — 종속 콤보·자동영문·자동코드·섹션 체크 + 저장(sections 포함) + 디자인.
3. **서버 site_code 정합** — 사업관리 자동생성(후속) + 별칭 조회(`findBySiteCode` 확장).
4. **마이그레이션** — 기존 site_code(gangjin 등) → 신코드 매핑/별칭 적재.

## 8. 검증 계획

- GUI: 종속 드롭다운·자동 영문/코드·섹션 저장 → 생성된 `site.{code}.json` 내용 확인(에이전트 dry-run, **시뮬레이션**).
- 서버: 신코드 payload → `findBySiteCode` 매칭 + 별칭 조회 단위/통합 테스트.
- codes.json 추출 정확성(강진 46810_UPIS 등).
- **지자체 현장 실측 수집은 사용자가 직접 진행**(분담 확정). codex 전체검증은 속도모드 종료 후.
