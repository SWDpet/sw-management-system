1. 배경 / 목표 WHY

현재 setup GUI 사이트 정보가 전부 자유 입력(지자체명·시스템명·사이트코드)이라:

  • 사이트 코드(gangjin)가 임의 → 같은 서버에 UPIS+다른 시스템 공존 시 코드 충돌 (site.{code}.json 중복)
  • 표기 불일치 → 서버 site_code 매핑 실패(422)

목표: ① 선택만으로 코드 자동조합(충돌 불가) ② 수집 섹션 선택(Step5 연동) ③ GUI 개편 ④ 폐쇄망 → codes.json 번들

2. 확정 결정 DECIDED

#항목확정
D1코드 형식{행정코드5}_{시스템영문} (시도2+시군구3 + _ + sys_nm_en)
D2폐쇄망 목록패키지에 codes.json 번들 동봉 (sigungu_code + sys_mst 추출)
D3섹션 기본값전부 체크(AP/DB/DBMS/GIS) 후 가감
D5기존 코드신코드 전환 + 별칭 유지
D4서버 매핑신코드를 sw_pjt.site_code 에 정합 (§6)
근거(DB 확인): adm_sect_c 전 행 5자리(강진 46810 · 단양 43800) · sys_mst(cd=영문 UPIS / nm=한글 도시계획정보체계) · sw_pjt.site_code VARCHAR(32) UNIQUE · 서버 findBySiteCode(payload.site) 미스 시 422.

3. 사이트 코드 규칙 CODE

site_code = adm_sect_c + "_" + sys_nm_en       (시스템영문 대문자, 행정코드 5자리)

강진군 UPIS = 46810_UPIS    강진군 KRAS = 46810_KRAS    단양군 UPIS = 43800_UPIS
  • 충돌 해소: 같은 PC라도 시스템영문이 달라 코드 분리 → 파일 중복 없음
  • 길이 ≤16자 → VARCHAR(32) 안전. payload.site = 파일명 = sw_pjt.site_code 동일 규칙 → end-to-end 일치

4. 폐쇄망 코드 번들 codes.json

빌드/배포 시 운영 DB 에서 추출해 에이전트에 동봉. setup GUI 는 이 파일만 읽음(인터넷·DB 불필요).

{
  "generated_at": "2026-05-31",
  "sigungu": [
    { "sido":"전라남도", "sgg":"강진군", "adm":"46810" },
    { "sido":"충청북도", "sgg":"단양군", "adm":"43800" }
  ],
  "systems": [ {"ko":"도시계획정보체계","en":"UPIS"}, {"ko":"부동산종합공부시스템","en":"KRAS"} ]
}

추출 스크립트 tools/export-codes.ps1. 시군구는 시도로 필터(종속). 시스템은 ko 표시/en 저장. 갱신=재배포 시.

7. 개발 단계 STEPS

  1. codes.json 추출 — export 스크립트 + 패키지 동봉(폐쇄망)
