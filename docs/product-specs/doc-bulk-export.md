- **상태**: v0.2 (codex 1차 반영 — 사용자 최종승인 대기)
- **작성일**: 2026-06-10 (v0.1) / v0.2 동일자 codex 반영
- **스프린트명**: `doc-bulk-export`
- **요청자**: 박욱진 — "핀/작성자로 거른 목록 기준, 기성계면 공문·검사원·기성내역서·일괄을 한 번에. 검색된 목록 전체 공문 다운로드 식으로. 다른 유형도 동일."

## 0. 한 줄 요약

사업문서 목록의 **현재 검색필터(문서유형·시도·시군구·작성자·키워드, 핀 포함)로 걸러진 전체 문서**에 대해, 선택한 **산출물 유형(공문/검사원/기성내역서/… 또는 전체)** 을 일괄 생성해 **ZIP** 으로 내려받는다.

## 1. 배경

- 산출물은 현재 **문서 1건씩** document-detail 에서 다운로드(공문/검사원/내역서/단건 ZIP `/document/api/zip/{id}`).
- 매월 기성 등 **수십 건**을 처리할 때 1건씩 받기가 비효율 → "거른 목록 전체"를 한 번에 받고 싶음.
- 검색 인프라(핀·작성자 포함)와 산출물 생성(docId 기반)·단건 ZIP·필터 일괄(excel-list) 선례가 이미 있음 → 결합만 하면 됨.

## 2. 범위 (In/Out)

**In**
- 사업문서 목록에서 **현재 필터 그대로** 전체(페이지네이션 무시) 조회 → 각 문서 산출물 생성 → ZIP 다운로드.
- 산출물 유형 선택:
  - **공문 일괄**(모든 유형 공통)
  - **전체 일괄**(각 문서의 유형별 전체 = 단건 ZIP 을 문서마다)
  - 유형 특화(문서유형 필터 선택 시): 기성=기성검사원/기성내역서, 착수=착수계본문/설계내역서, 준공=준공계본문
- ZIP 내 파일명: 문서별 유일(시군구·사업명·문서번호 기반).

**Out**
- 비동기/백그라운드 작업 큐(이번은 동기 + 건수 상한).
- 이메일 발송·외부 저장.
- PDF 일괄(필요 시 후속).

## 3. 기능 요구사항 (FR)

| ID | 요구사항 |
|---|---|
| FR-1 | 목록 화면에 **"일괄 산출물 다운로드 ▾"** 컨트롤. 현재 필터(FormData, excel-list 와 동일 방식)를 쿼리로 전달. |
| FR-2 | 신규 엔드포인트 `GET /document/api/bulk-zip?type={유형}&{필터파라미터}` — 같은 필터로 전체 조회 → 각 docId 산출물 생성 → ZIP 반환. |
| FR-3 | `type` 값: `letter`(공문) / `all`(전체) / `inspector`(기성검사원) / `interim`(기성내역서) / `commence_body`(착수본문) / `design`(설계내역서) / `completion`(준공본문). |
| FR-4 | **유형 불일치 문서는 건너뜀**: 예) `inspector` 일괄을 혼합 목록에 돌리면 기성계 문서만 생성. 건너뛴 건수 안내(파일명/요약). `letter`·`all` 은 각 문서의 유형에 맞춰 생성. |
| FR-5 | ZIP 파일명: `{유형라벨}_일괄_{날짜}.zip`. 내부 파일명: `{시군구}_{사업명}_{문서번호}_{산출물}.{ext}` (중복 방지: 충돌 시 _2 등 suffix). |
| FR-6 | **전체 일괄(`all`)** 은 문서마다 그 유형의 전체 산출물(단건 ZIP 구성)을 ZIP 안에 — **문서별 하위폴더**(`{시군구}_{사업명}/...`)로 정리. |
| FR-7 | 권한: bulk-zip 진입 시 **`getAuth() == NONE` 명시 차단**(403). 단건 hwpx/excel/zip 은 권한체크가 약하므로 벌크는 반드시 명시(codex). |
| FR-8 | **건별 try-catch**(실패 문서 로그+건너뜀, 나머지 ZIP 진행). 실패·건너뜀(유형 불일치, 준공 KRAS/UPIS 조용 실패 포함)을 ZIP 내 **`_실패목록.txt`** 에 기록(codex). |
| FR-9 (codex) | **건수 상한 200**. `totalElements > 200` 이면 ZIP 생성 전 **차단**(413 + "검색 결과 N건 — 최대 200건, 필터를 좁히세요"). 조용한 truncation 금지. |
| FR-10 (codex) | 대상 **0건이면 400** "대상 문서 없음". |
| FR-11 (codex) | ZIP 엔트리(폴더/파일)명 **sanitize 필수** — 경로 금지문자(`\ / : * ? " < > \| ..`) 제거(`DocumentSignedScanService.sanitize` 유사). raw 사용 금지(경로 깨짐·보안). |

## 4. 설계

### 4-1. 엔드포인트 (DocumentController)
- `GET /document/api/bulk-zip` — `excel-list`(158-192) 와 동일 필터 파라미터(docType·cityNm·distNm·keyword·authorName) + `type`.
- `searchDocuments(filter, PageRequest.of(0, 한도))` 로 전체 DocumentDTO 조회.
- 각 DTO 의 docId·docType 기준 산출물 생성:
  - `letter` → `generateHwpx(docId, "letter")`
  - `inspector` → 기성계만 `generateHwpx(docId, "inspector")`
  - `interim` → 기성계만 `generateInterimReport(docId)`
  - `commence_body` → 착수계만 / `design` → 착수계 `generateDesignEstimate` / `completion` → 준공계 `generateHwpx(docId, "completion_body")`
  - `all` → 각 문서 유형의 단건 ZIP 구성 그대로(공문+유형별)
- `java.util.zip.ZipOutputStream`(단건 ZIP `584-621` 패턴 재사용)으로 묶어 `byte[]` 반환. 파일명 UTF-8 인코딩(기존 패턴).

### 4-2. 산출물 생성 재사용
- 기존 docId 기반 메서드(`HwpxExportService.generateHwpx`, `ExcelExportService.generateInterimReport/generateDesignEstimate`) 그대로 반복 호출. **신규 생성 로직 없음**(조립만).

### 4-3. 파일명/중복
- 같은 시군구·동일 사업명 충돌 대비 문서번호 포함 + 최종 충돌 시 `_n` suffix. ZIP 엔트리명 정제(경로 금지문자 제거 — doc-signed-scan `sanitize` 유사).

## 5. DB팀 가상 자문
- **스키마 변경 없음**. 기존 검색 + docId 기반 생성 재사용. 추가 쿼리 없음(searchDocuments 그대로).

## 6. 디자인팀 가상 자문 (A+D — UI 변경)
- **배치**: 필터바의 `엑셀 다운로드` 옆에 **"일괄 산출물 ▾"** 드롭다운(`.dropdown` 기존 패턴 = 문서작성 버튼과 동일 컴포넌트 재사용 → 일관성).
- **옵션 동적**: 선택된 문서유형에 따라 메뉴 구성.
  - 기성계: 공문 일괄 / 기성검사원 일괄 / 기성내역서 일괄 / **전체 일괄**
  - 착수계: 공문 / 착수계본문 / 설계내역서 / 전체
  - 준공계: 공문 / 준공계본문 / 전체
  - 유형 미선택(전체): **공문 일괄 / 전체 일괄**만(공통). 유형 특화는 비활성+안내.
- **토큰·다크모드**: 기존 `.dropdown-content`·`--primary` 토큰 재사용, 신규 색 없음.
- **접근성**: 드롭다운 `aria-haspopup`/`aria-expanded`, 항목 키보드 접근, 다운로드 진행 안내(건수 많으면 시간 소요 → 클릭 시 "생성 중" 토스트/비활성).
- **피드백**: 대량이면 수십 초 걸릴 수 있으니 버튼 비활성+스피너(브라우저 다운로드 시작까지).

## 7. 비기능 / 리스크
- **성능/규모**: N문서 × 산출물 동기 생성 → 대량(수백)일 때 수십 초~분. **건수 상한**(예: 200) + 초과 시 안내(필터 좁히기). excel-list 는 100000 이지만 산출물 생성은 무거우므로 별도 상한.
- **타임아웃**: 동기 응답이라 서버/프록시 타임아웃 고려(현재 nginx proxy_read_timeout 300s — §운영). 상한으로 관리.
- **메모리**: 산출물 byte[] 다수 ZIP → 큰 목록 시 힙. 스트리밍 ZIP(ZipOutputStream 에 바로 write) 으로 누적 최소화.
- **부분 실패**: 건별 try-catch(FR-8), 실패 목록 ZIP 내 `_실패목록.txt` 또는 응답 헤더 안내.

## 8. 결정 사항 (codex 반영 — 확정안, 사용자 확인 요청)
1. **전체 일괄(`all`) ZIP 구조** = **문서별 하위폴더**: `{시군구}_{사업명}_{문서번호}/공문.hwpx`, `/기성내역서.xlsx` … (중첩 ZIP 아님 — UX·구현 모두 유리, codex).
2. **건수 상한 = 200**, 초과 시 **차단**(413). (조용한 truncation 금지.)
3. **유형 미선택(혼합) 목록**: `letter`(공문)·`all`(전체)만 노출. 유형특화(검사원/내역서 등)는 문서유형 필터 선택 시에만 활성.
4. **파일명** = `{시군구}_{사업명}_{문서번호}_{산출물}.{ext}` (sanitize 후, 충돌 시 `_n`).

> 위 4개는 codex 권고 기반 기본안. 다른 선호 있으면 승인 시 알려주세요(특히 ② 상한 200, ① 하위폴더 구조).
